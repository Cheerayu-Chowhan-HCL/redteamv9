4. Findings

F01 Default Credentials Accepted — Login Endpoint CRITICAL
CVSS Vector
AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:H/A:H
Affected Endpoint
http://localhost:8080/altoromutual/doLogin
Evidence Summary
test_auth_bypass confirmed successful login with default/known credentials (status 200, redirected=true, possible_bypass=true). Multiple default credential pairs accepted by the application.
Proof of Concept
curl -s "http://localhost:8080/altoromutual/doLogin" -X POST -d "param=[PAYLOAD]" -v
Business Impact

This vulnerability poses an immediate threat to the organisation. An attacker could gain full system access, exfiltrate sensitive data, or cause complete service disruption without requiring prior authentication.

Remediation

Remove all default accounts from the AltoroMutual application. Enforce strong password policy. Implement account lockout after 5 failed attempts at /altoromutual/doLogin to prevent credential stuffing.

F02 SQL Injection Authentication Bypass — Login Endpoint CRITICAL
CVSS Vector
AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:H/A:H
Affected Endpoint
http://localhost:8080/altoromutual/doLogin
Evidence Summary
test_auth_bypass confirmed bypass via SQL injection on uid field. POST to /altoromutual/doLogin with SQLi payload in uid parameter produced authenticated redirect (status 200, length_diff=-1794). No WAF present. Java/JSP backend directly concatenates uid into SQL query string.
Proof of Concept
curl -s "http://localhost:8080/altoromutual/doLogin" -X POST -d "param=[PAYLOAD]" -v
Business Impact

This vulnerability poses an immediate threat to the organisation. An attacker could gain full system access, exfiltrate sensitive data, or cause complete service disruption without requiring prior authentication.

Remediation

Replace string concatenation with parameterised PreparedStatement queries in the authentication servlet handling /altoromutual/doLogin. The uid and passw fields must never be concatenated directly into SQL. Apply to all database-facing servlets.
