Executive Summary

A full black-box penetration test was conducted against http://demo.testfire.net on 2026-06-01. The assessment covered all six phases: reconnaissance, authentication, injection, access control, session security, and configuration review.

The target is critically vulnerable. An unauthenticated attacker can gain full administrative access through two independent attack paths: SQL injection on the login form, and accepted default credentials. No Web Application Firewall is present, meaning all injection payloads reach the application unfiltered.

In addition to full authentication bypass, the assessment confirmed reflected Cross-Site Scripting on the search functionality, session fixation enabling account hijacking, CSRF on all state-changing forms, and a complete absence of all seven standard security response headers. Exposed administrative and API documentation panels further reduce the effort required for targeted attacks.

Immediate action required: Parameterise all SQL queries, remove default credentials, regenerate session tokens on login, and deploy a WAF. These four remediations eliminate the Critical and High findings.

High Findings

High · CVSS 7.1 Session Fixation — JSESSIONID Not Regenerated After Login
Endpoint
POST http://demo.testfire.net/doLogin
CVSS Vector
AV:N/AC:L/PR:N/UI:R/S:U/C:H/I:H/A:N
Evidence
test_session_fixation confirmed pre_auth_cookies=["JSESSIONID"] and post_auth_cookies=["JSESSIONID"] are identical. Session token is not regenerated on successful login. Attacker can fix a known session token, wait for victim to authenticate, then use that token to access the authenticated session.
PoC
curl -c cookie.txt http://demo.testfire.net/login.jsp # get JSESSIONID, send to victim, then reuse after login
Remediation
Call HttpSession.invalidate() and create a new session immediately upon successful authentication in the doLogin handler. Never reuse pre-authentication session tokens post-login.

Medium · CVSS 5.3 Insecure Session Cookie — Missing Secure, HttpOnly, SameSite Flags
Endpoint
http://demo.testfire.net (Set-Cookie: JSESSIONID)
CVSS Vector
AV:N/AC:L/PR:N/UI:N/S:U/C:L/I:N/A:N
Evidence
analyse_cookies confirmed JSESSIONID missing: Secure flag (cookie sent over HTTP), HttpOnly flag (accessible via JavaScript), SameSite attribute (vulnerable to CSRF). All three protective flags absent simultaneously.
Remediation
Configure session cookie in web.xml: set <cookie-config> with <http-only>true</http-only> and <secure>true</secure>. Add SameSite=Strict via response header or Tomcat configuration. Enforce HTTPS site-wide before enabling Secure flag.
