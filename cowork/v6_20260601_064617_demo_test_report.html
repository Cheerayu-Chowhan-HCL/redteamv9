F01 Default Credentials — admin/admin grants access to /bank/main.jsp CRITICAL
CVSS Vector
AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:H/A:H
Affected Endpoint
http://demo.testfire.net/doLogin
Evidence Summary
sqlmap POST with uid=admin&passw=admin produced HTTP 302 redirect to /bank/main.jsp (authenticated banking portal). Default credentials admin/admin are valid and grant full application access.
Proof of Concept
curl -s "http://demo.testfire.net/doLogin" -X POST -d "param=[PAYLOAD]" -v
Business Impact

This vulnerability poses an immediate threat to the organisation. An attacker could gain full system access, exfiltrate sensitive data, or cause complete service disruption without requiring prior authentication.

Remediation

Immediately change or disable the default admin account at /doLogin. Enforce a strong password policy. Audit all user accounts in the Apache Derby database for weak or default credentials. Implement account lockout after failed attempts.

F02 SQL Injection — POST parameter uid at /doLogin (Apache Derby, Boolean-Based Blind) CRITICAL
CVSS Vector
AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:H/A:H
Affected Endpoint
http://demo.testfire.net/doLogin
Evidence Summary
sqlmap confirmed: POST parameter uid is AND boolean-based blind injectable. Backend DBMS fingerprinted as Apache Derby. Payload: uid=[PAYLOAD]&passw=admin&btnSubmit=Login. Server follows 302 redirect to /bank/main.jsp on successful authentication demonstrating live database interaction.
Proof of Concept
curl -s "http://demo.testfire.net/doLogin" -X POST -d "param=[PAYLOAD]" -v
Business Impact

This vulnerability poses an immediate threat to the organisation. An attacker could gain full system access, exfiltrate sensitive data, or cause complete service disruption without requiring prior authentication.

Remediation

Replace string concatenation with parameterised PreparedStatement queries at the doLogin servlet. The uid field is directly concatenated into the SQL authentication query. Use: PreparedStatement ps = conn.prepareStatement("SELECT * FROM users WHERE uid=? AND passw=?"); ps.setString(1, uid); ps.setString(2, passw). Never pass user input directly into SQL string construction.
